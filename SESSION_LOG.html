Step 3 — Supabase Schema

[Bash: mkdir -p supabase] [Write: supabase/schema.sql]

Created supabase/schema.sql with both tables from Section 4 of the spec:

  • posts — id (uuid), title, slug (unique), content, excerpt, category (enum check: ai-models/anthropic/industry/tools), tags (text[]), cover_image_url, cover_image_prompt, source_urls (text[]), status (enum check: published/draft), auto_generated, created_at, published_at. Indexes on (status, published_at), category, slug.
  • scout_memory — id, feed_url, item_guid, item_title, processed_at, post_id (FK → posts.id, nullable). Unique constraint on (feed_url, item_guid) to prevent duplicates. Index on (feed_url, item_guid).

Row Level Security enabled on both tables. Public policy allows SELECT on published posts only. Service role key bypasses RLS by default — no extra policy needed for Scout writes.

Step 4 — API Routes

[Write: lib/supabase.ts — lazy proxy Supabase client (public + admin)] [Write: lib/slugify.ts — slug generation utility] [Write: lib/types.ts — Post TypeScript interface] [Write: lib/createPost.ts — shared post-creation logic with unique slug generation] [Write: app/api/health/route.ts — GET returns { status: "ok" }] [Write: app/api/posts/create/route.ts — POST, x-api-key auth, publishes post] [Write: app/api/posts/draft/route.ts — POST, x-api-key auth, saves as draft]

All three API routes check the x-api-key header against process.env.SCOUT_API_KEY. The /api/posts/create and /api/posts/draft routes share the same validation and createPost() logic — only the status argument differs.

The slug generation in lib/createPost.ts follows the spec exactly:

  • Lowercase, hyphen-separated, special chars stripped, max 80 chars
  • Appends -2, -3 etc. if slug already exists (loop queries Supabase until unique)

The /api/posts/create response matches the spec:

{ success: true, post_id: "uuid", slug: "...", url: "https://aitrends.ng/post/..." }

The Supabase client in lib/supabase.ts uses a lazy Proxy pattern. The first approach (module-level createClient() call) caused the build to fail with supabaseUrl is required during next build when env vars are empty. The Proxy defers instantiation to the first method call, so the module can be imported safely at build time without env vars present.
